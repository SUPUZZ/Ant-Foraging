# 蚂蚁觅食仿真 — 行为规律（与 `ant-foraging.html` 一致）

本文档描述当前网页仿真中的**全部规则**，便于对照代码核对或修改。仿真为二维连续平面，信息素在离散网格上累加；画布上的颜色仅用于**显示**信息素强度，不改变数值仿真。

---

## 1. 世界与离散化

| 项目 | 取值 |
|------|------|
| 画布逻辑尺寸 | 宽 `W = 720`，高 `H = 520`（由 `<canvas id="sim">` 的 `width` / `height` 决定） |
| 信息素网格边长 | `CELL = 4`（像素） |
| 横向格数 | `GW = ceil(W / CELL)` |
| 纵向格数 | `GH = ceil(H / CELL)` |
| 格索引 | 世界坐标 `(x, y)` 映射到格 `(cx, cy) = (floor(x/CELL), floor(y/CELL))`，格内信息素为该格标量累加值 |

任意格子可通过一维数组下标 `i = cy * GW + cx` 访问（越界则该处采样为 0、沉积无效）。

---

## 2. 实体定义

### 2.1 巢穴

- 位置：画布中心 `(nest.x, nest.y) = (W * 0.5, H * 0.5)`。
- 半径：`nest.r = 14`（像素）。

### 2.2 食物源

每个食物源为 `{ x, y, r, amount }`：

- 碰撞半径：`r = 10`。
- 初始数量：调用 `addFood` 时若未指定，默认 `amount = 80`。
- 「重置场景」时生成的随机食物：`amount` 在区间 `[60, 140]` 上均匀随机（即 `60 + random * 80`）。
- 用户点击画布添加：`amount = 100`（且点击位置须与巢穴保持一定距离，见第 5 节）。

### 2.3 蚂蚁

每只蚂蚁为 `{ x, y, ang, state }`：

- `state === 0`：**觅食**。
- `state === 1`：**返巢**（携带食物的路上）。

出生：在巢穴边缘附近、`nest.r * 0.5` 距离内随机角度放置；初始朝向 `ang` 为 `[0, 2π)` 均匀随机。

---

## 3. 状态机与边界

每帧对每只蚂蚁依次更新（顺序固定为数组遍历顺序）。

### 3.1 觅食 → 返巢

若当前为觅食，且存在某食物满足 `amount > 0` 且欧氏距离  
`dist(蚂蚁, 食物) < 食物.r + 4`（即 `< 14` 像素），则该食物 `amount` 减 1，蚂蚁 `state` 置为 `1`（仅处理碰到的第一个满足条件的食物）。

### 3.2 返巢 → 觅食

若当前为返巢，且 `dist(蚂蚁, 巢心) < nest.r + 2`（即 `< 16` 像素），则：

- 全局计数「累计运回」加 1；
- 蚂蚁重新变为觅食：在距巢心 `nest.r * 0.4` 的圆上随机一点重生，朝向设为 `spawnA + π + 小幅随机`。

### 3.3 硬边界（非环绕）

移动后坐标被**夹紧**在 `[2, W-2]` × `[2, H-2]`（避免贴边数值问题），不是圆环拓扑上的环绕。

---

## 4. 信息素

- **沉积**：仅当蚂蚁处于返巢状态 `state === 1` 时，在其当前世界坐标所在格累加  
  `Δ = pStr × 0.35`  
  其中 `pStr` 为界面滑块「信息素强度」（默认 12，范围约 `[1, 40]`）。
- **挥发**：每帧**先**对所有格执行 `pher[i] *= evap`，其中 `evap` 为界面「信息素衰减」（默认 `0.992`，可调）。

因此单帧顺序为：全局挥发 → 蚂蚁运动与沉积（沉积在挥发之后发生）。

---

## 5. 运动规则

以下为界面可调参数：`speed`（移动速度）、`noise`（觅食随机性）。返巢时还会用到上述信息素参数。

### 5.1 返巢（`state === 1`）

1. 在当位置按第 4 节沉积信息素。
2. 设 `toNest = atan2(nest.y - y, nest.x - x)`，更新朝向  
   `ang += (toNest - ang) * 0.22 + (random - 0.5) * noise * 0.5`。
3. 位移：  
   `x += cos(ang) * speed * 1.15`，`y += sin(ang) * speed * 1.15`，再经第 3.3 节边界夹紧。

### 5.2 觅食（`state === 0`）

常量（写死在代码中）：

- 感应距离 `senseDist = 14`；
- 左右感应相对航向角 `senseAng = 0.55`（弧度）；
- 信息素引导门限 `0.02`（仅当 `max(左, 中, 右) > 0.02` 时按梯度转向）。

在航向的左、中、右三个方向，于距离 `senseDist` 处各采样一格信息素标量，记为 `pl, pc, pr`。

- 基础转角：`turn = (random - 0.5) * noise * 2`。
- 若 `max(pl, pc, pr) > 0.02`：  
  - 若右侧最强（`pr >= pl` 且 `pr >= pc`），`turn += senseAng * 0.55`；  
  - 否则若左侧最强（`pl >= pr` 且 `pl >= pc`），`turn -= senseAng * 0.55`。  
  （中间独强时不额外偏转。）
- `ang += turn`。
- 位移：`x += cos(ang) * speed`，`y += sin(ang) * speed`，再边界夹紧。

## 6. 界面与交互（与规则对应）

| UI | 代码变量 | 含义 |
|----|----------|------|
| 蚂蚁数量 | `ants` | 重置时初始化；滑动条变更时增减蚂蚁数组长度（新蚂蚁用 `makeAnt()`） |
| 移动速度 | `speed` | 觅食与返巢位移的标量系数（返巢再乘 `1.15`） |
| 信息素强度 | `pher` → `pStr` | 沉积量乘数，见第 4 节 |
| 信息素衰减 | `evap` | 每帧全局挥发系数 |
| 觅食随机性 | `noise` | 觅食转角与返巢朝向扰动 |
| 重置场景 | — | 清空信息素、重新随机散布 **5** 处食物、累计运回归零、按当前蚂蚁数量重建 |
| 随机散布食物 | — | 追加 **4** 处随机食物（数量分布同重置时的随机食物） |
| 点击画布 | — | 若点击点与巢心距离 `> nest.r + 6`，则在点击处 `addFood(x, y, 100)` |

统计栏：「场上食物」为各食物 `amount` 向下取整后求和；「返巢中 / 觅食中」为当前状态计数。

---

## 7. 可视化（不参与仿真）

- 信息素：每个格用 `maxShow = 18` 将归一化强度映射到 RGB（青绿—蓝色调），仅 `fillRect(CELL × CELL)` 铺色。
- 巢穴：半透明圆；食物：半径随 `amount` 变化，公式 `半径 = 4 + min(10, amount / 25)`（显示用）。
- 蚂蚁：三角形；返巢时叠加浅橙色半透明圆表示携带状态。

修改本节映射或颜色**不会改变**第 2—5 节的仿真数值。

---

## 8. 未实现的扩展

当前版本**不包含**：障碍物、多种信息素类型、蚁群繁殖或死亡、精确物理碰撞体等。若需加入，应在本文档增补对应条目后再改代码。
